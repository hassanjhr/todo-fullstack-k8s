## Summary

Deploy the existing Todo Full-Stack app with all **5 Dapr building blocks** (Pub/Sub, State, Bindings/Cron, Secrets, Service Invocation) on:
- **Part B**: Minikube — fully automated via Helm chart with `dapr.enabled=true`

No new database tables. Changes are: Dapr component YAMLs, Helm chart updates, a new backend cron binding handler, frontend Dapr service invocation wiring, GitHub Actions workflow, and a monitoring stack.

## Implementation Phases

### Phase 1: Complete All 5 Dapr Components (Helm + Local)

**Goal**: All 5 Dapr building blocks configured and working on both Minikube and local `dapr run`.

#### 1.1 — Dapr State Store component

Add to `todo-chatbot/templates/dapr-components.yaml` (inside `{{- if .Values.dapr.enabled }}`):

```yaml
---
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: statestore
  namespace: {{ .Release.Namespace }}
spec:
  type: state.redis
  version: v1
  metadata:
    - name: redisHost
      value: "redis-master:6379"    # dapr init --kubernetes installs redis-master
    - name: redisPassword
      value: ""
    - name: actorStateStore
      value: "true"
  scopes:
    - todo-backend
```

Add `dapr/components/statestore.yaml` for local `dapr run`:
```yaml
spec:
  type: state.redis
  version: v1
  metadata:
    - name: redisHost
      value: "localhost:6379"    # dapr init (local) installs Redis on localhost
```

#### 1.2 — Dapr Cron Binding component

Add to `todo-chatbot/templates/dapr-components.yaml`:

```yaml
---
apiVersion: dapr.io/v1alpha1
kind: Component
metadata:
  name: reminder-cron
  namespace: {{ .Release.Namespace }}
spec:
  type: bindings.cron
  version: v1
  metadata:
    - name: schedule
      value: {{ .Values.dapr.cronSchedule | default "@every 5m" | quote }}
  scopes:
    - todo-backend
```

Add `dapr/components/reminder-cron.yaml` for local:
```yaml
spec:
  type: bindings.cron
  version: v1
  metadata:
    - name: schedule
      value: "@every 5m"
```

#### 1.3 — Backend cron binding handler

New file: `backend/src/api/routes/dapr_bindings.py`

```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from src.api.deps import get_db
from src.services.reminder_service import process_due_reminders

router = APIRouter()

@router.post("/reminder-cron")
async def handle_reminder_cron(db: AsyncSession = Depends(get_db)):
    """Dapr cron binding handler — fires on schedule to process due reminders."""
    await process_due_reminders(db)
    return {"status": "ok"}
```

Register in `backend/src/main.py`:
```python
from src.api.routes.dapr_bindings import router as bindings_router
app.include_router(bindings_router)
```

#### 1.4 — Service Invocation: Dapr sidecar on frontend pod

Update `todo-chatbot/templates/deployment.yaml` — add Dapr annotations to frontend pod:

```yaml

```

Update `todo-chatbot/values.yaml`:
```yaml

  enabled: false
  cronSchedule: "@every 5m"
  enableMetrics: true
```

Update `frontend/.env.local` (and frontend Helm env) to support Dapr service invocation:

Add `frontend/lib/api/client.ts` utility that switches base URL based on `NEXT_PUBLIC_DAPR_ENABLED`.

### Phase 3: GitHub Actions CI/CD

**Goal**: Push to `main` → build → push GHCR → deploy to OKE automatically.

#### 3.1 — `.github/workflows/deploy.yml`

```yaml
name: Build & Deploy to OKE

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  BACKEND_IMAGE: ghcr.io/${{ github.repository_owner }}/todo-backend
  FRONTEND_IMAGE: ghcr.io/${{ github.repository_owner }}/todo-frontend

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    outputs:
      image_tag: ${{ github.sha }}
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/build-push-action@v5
        with:
          context: ./backend
          push: true
          tags: |
            ${{ env.BACKEND_IMAGE }}:latest
            ${{ env.BACKEND_IMAGE }}:${{ github.sha }}
      - uses: docker/build-push-action@v5
        with:
          context: ./frontend
          push: true
          tags: |
            ${{ env.FRONTEND_IMAGE }}:latest
            ${{ env.FRONTEND_IMAGE }}:${{ github.sha }}

  deploy:
    needs: build-and-push
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up kubectl
        run: |
          echo "${{ secrets.OKE_KUBECONFIG }}" | base64 -d > kubeconfig.yaml
          export KUBECONFIG=kubeconfig.yaml
      - name: Install Dapr CLI
        run: |
          wget -q https://raw.githubusercontent.com/dapr/cli/master/install/install.sh -O - | /bin/bash
      - name: Init Dapr on OKE (idempotent)
        run: |
          export KUBECONFIG=kubeconfig.yaml
          dapr init --kubernetes --wait || dapr upgrade --kubernetes --wait
      - name: Deploy with Helm
        run: |
          export KUBECONFIG=kubeconfig.yaml
          helm upgrade --install todo-chatbot ./todo-chatbot \
            --set dapr.enabled=true \
            --set backend.image.repository=${{ env.BACKEND_IMAGE }} \
            --set backend.image.tag=${{ github.sha }} \
            --set backend.image.pullPolicy=Always \
            --set frontend.image.repository=${{ env.FRONTEND_IMAGE }} \
            --set frontend.image.tag=${{ github.sha }} \
            --set frontend.image.pullPolicy=Always \
            --set frontend.service.type=LoadBalancer \
            --set env.DATABASE_URL="${{ secrets.DATABASE_URL }}" \
            --set env.JWT_SECRET_KEY="${{ secrets.JWT_SECRET_KEY }}" \
            --set env.OPENAI_API_KEY="${{ secrets.OPENAI_API_KEY }}"
      - name: Verify rollout
        run: |
          export KUBECONFIG=kubeconfig.yaml
          kubectl rollout status deployment/todo-chatbot-backend --timeout=5m
          kubectl rollout status deployment/todo-chatbot-frontend --timeout=5m
```

## Risk Analysis

| Risk | Blast Radius | Mitigation |
|------|-------------|------------|
| OKE ARM image incompatibility | Deployment fails | Verify ARM64 manifests in research (all confirmed available) |
| Dapr cron binding double-fires | Duplicate reminders sent | Make `process_due_reminders` idempotent (check `is_sent` flag already in DB) |
| OKE Load Balancer not provisioned (quota) | Frontend unreachable | Fall back to NodePort + `kubectl port-forward` |
| `dapr init --kubernetes` in CI fails on already-installed Dapr | CI blocked | Use `|| dapr upgrade` fallback (in workflow) |
| GitHub Actions OKE kubeconfig rotates | Deploy fails | Store kubeconfig as secret, document rotation procedure |
